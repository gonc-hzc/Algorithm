# 线性表

## 1. 线性表的定义

线性表是由 `n` 个数据类型相同的数据元素组成的有限序列。

通常记作：

```text
L = (a1, a2, ..., ai, ..., an)
```

其中：

- `L` 表示线性表。
- `n` 表示线性表的长度。
- `n = 0` 时，称为空表。
- `ai` 表示线性表中的第 `i` 个元素。
- `a1` 是第一个元素，也叫表头元素。
- `an` 是最后一个元素，也叫表尾元素。

线性表的核心特点是：元素之间存在一对一的线性关系。

## 2. 线性表的逻辑特征

在线性表中：

- 除第一个元素外，每个元素有且只有一个直接前驱。
- 除最后一个元素外，每个元素有且只有一个直接后继。
- 第一个元素没有直接前驱。
- 最后一个元素没有直接后继。

例如：

```text
L = (10, 20, 30, 40)
```

对于元素 `30`：

- 直接前驱是 `20`。
- 直接后继是 `40`。

## 3. 位序与下标

学习线性表时，要注意区分位序和数组下标。

| 概念 | 起始值 | 示例 |
| --- | --- | --- |
| 位序 | 从 `1` 开始 | 第 1 个元素、第 2 个元素 |
| 下标 | 从 `0` 开始 | `data[0]`、`data[1]` |

如果第 `i` 个元素存放在数组中，则它对应的数组下标是：

```text
i - 1
```

例如：

```text
第 1 个元素 -> data[0]
第 2 个元素 -> data[1]
第 i 个元素 -> data[i - 1]
```

## 4. 线性表的基本操作

线性表常见基本操作可以概括为：增、删、改、查。

### 4.1 初始化

创建一个空的线性表。

```text
InitList(&L)
```

作用：

- 分配或准备存储空间。
- 将线性表长度设置为 `0`。

### 4.2 求表长

```text
Length(L)
```

作用：返回线性表中元素的个数。

### 4.3 判断空表

```text
IsEmpty(L)
```

作用：判断线性表是否为空。

### 4.4 按位查找

```text
GetElem(L, i)
```

作用：查找线性表中第 `i` 个位置的元素。

### 4.5 按值查找

```text
LocateElem(L, e)
```

作用：查找线性表中第一个值等于 `e` 的元素位置。

### 4.6 插入

```text
ListInsert(&L, i, e)
```

作用：在线性表第 `i` 个位置插入元素 `e`。

插入前：

```text
L = (a1, a2, ..., ai, ..., an)
```

插入后：

```text
L = (a1, a2, ..., e, ai, ..., an)
```

### 4.7 删除

```text
ListDelete(&L, i, &e)
```

作用：删除线性表第 `i` 个位置的元素，并用 `e` 返回被删除的值。

删除前：

```text
L = (a1, a2, ..., ai, ai+1, ..., an)
```

删除后：

```text
L = (a1, a2, ..., ai-1, ai+1, ..., an)
```

### 4.8 遍历

```text
PrintList(L)
```

作用：依次访问线性表中的每一个元素。

### 4.9 销毁

```text
DestroyList(&L)
```

作用：释放线性表占用的存储空间。

对于静态数组实现的顺序表，一般不需要手动释放；对于动态分配的顺序表和链表，通常需要释放内存。

## 5. 线性表的存储结构

线性表是一种逻辑结构，它描述的是元素之间的逻辑关系。

线性表在计算机中主要有两种存储方式：

- 顺序存储结构：顺序表。
- 链式存储结构：链表。

## 6. 顺序存储结构：顺序表

顺序表使用一段地址连续的存储单元依次存放线性表中的元素。

特点：

- 逻辑上相邻，物理位置也相邻。
- 可以通过下标直接访问元素。
- 插入和删除时通常需要移动元素。

静态顺序表结构：

```c
#define MAXSIZE 100

typedef int ElemType;

typedef struct
{
    ElemType data[MAXSIZE];
    int length;
} SeqList;
```

优点：

- 支持随机访问，按位查找时间复杂度为 `O(1)`。
- 存储密度高，不需要额外指针域。
- 实现简单。

缺点：

- 插入、删除需要移动元素，平均时间复杂度为 `O(n)`。
- 静态顺序表容量固定，可能溢出或浪费空间。

更多顺序表细节见：[顺序表.md](./顺序表.md)。

## 7. 链式存储结构：链表

链表使用一组不一定连续的存储单元存放线性表中的元素。

每个结点通常包含两部分：

- 数据域：存放数据元素。
- 指针域：存放后继结点的地址。

单链表结点结构：

```c
typedef int ElemType;

typedef struct LNode
{
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;
```

特点：

- 逻辑上相邻，物理位置不一定相邻。
- 通过指针把元素连接起来。
- 查找需要从头结点开始逐个访问。
- 插入和删除不需要大量移动元素，只需要修改指针。

优点：

- 插入、删除方便。
- 不要求存储空间连续。
- 可以按需申请和释放空间。

缺点：

- 不支持随机访问。
- 查找第 `i` 个元素需要从头遍历，时间复杂度为 `O(n)`。
- 每个结点需要额外的指针域，存储密度低于顺序表。

## 8. 顺序表和链表的比较

| 比较项目 | 顺序表 | 链表 |
| --- | --- | --- |
| 存储空间 | 连续 | 不一定连续 |
| 随机访问 | 支持，`O(1)` | 不支持，`O(n)` |
| 按值查找 | `O(n)` | `O(n)` |
| 插入删除 | 需要移动元素 | 只需修改指针 |
| 存储密度 | 高 | 较低 |
| 空间分配 | 通常预先分配 | 按需动态分配 |
| 适合场景 | 查找多、元素数量变化小 | 插入删除多、元素数量变化大 |

选择建议：

- 如果经常按位置访问元素，优先考虑顺序表。
- 如果经常插入和删除元素，优先考虑链表。
- 如果元素个数变化不大，顺序表更简单。
- 如果元素个数变化较大，链表更灵活。

## 9. 线性表操作复杂度总结

| 操作 | 顺序表 | 链表 |
| --- | --- | --- |
| 按位查找 | `O(1)` | `O(n)` |
| 按值查找 | `O(n)` | `O(n)` |
| 表头插入 | `O(n)` | `O(1)` |
| 表尾插入 | `O(1)` 或 `O(n)` | 有尾指针 `O(1)`，无尾指针 `O(n)` |
| 指定位置插入 | `O(n)` | 查找位置 `O(n)`，插入本身 `O(1)` |
| 表头删除 | `O(n)` | `O(1)` |
| 表尾删除 | `O(1)` | 单链表通常 `O(n)` |
| 指定位置删除 | `O(n)` | 查找位置 `O(n)`，删除本身 `O(1)` |
| 遍历 | `O(n)` | `O(n)` |

注意：

- 链表插入、删除本身只需要修改指针，是 `O(1)`。
- 但如果要先找到第 `i` 个位置，查找过程仍然是 `O(n)`。

## 10. 常见链表类型

### 10.1 单链表

每个结点只有一个 `next` 指针，指向后继结点。

```text
head -> a1 -> a2 -> a3 -> NULL
```

特点：

- 只能从前往后访问。
- 插入删除时需要知道前驱结点。

### 10.2 双链表

每个结点有两个指针：

- `prior` 指向前驱结点。
- `next` 指向后继结点。

```text
NULL <- a1 <-> a2 <-> a3 -> NULL
```

特点：

- 可以向前访问，也可以向后访问。
- 删除某个结点时更方便。
- 每个结点需要两个指针域。

### 10.3 循环链表

循环链表的最后一个结点不指向 `NULL`，而是指回头结点或第一个结点。

```text
head -> a1 -> a2 -> a3
 ^                  |
 |__________________|
```

特点：

- 从任意结点出发都可以访问整个链表。
- 常用于循环处理问题。

### 10.4 静态链表

静态链表用数组模拟链表。

结点中通常包含：

- `data`：数据域。
- `next`：游标，表示下一个元素在数组中的下标。

适用于不方便使用指针的场景。

## 11. 线性表常见算法思想

### 11.1 顺序扫描

从头到尾依次访问元素。

常用于：

- 按值查找。
- 统计元素个数。
- 查找最大值、最小值。
- 判断某个元素是否存在。

### 11.2 双指针

使用两个下标或指针从不同方向或不同速度遍历。

常用于：

- 顺序表逆置。
- 奇偶调整。
- 有序表合并。
- 链表找中间结点。
- 链表判断是否有环。

### 11.3 前驱指针

在链表中删除或插入结点时，常常需要保存当前结点的前驱。

例如删除单链表中值为 `x` 的结点时，需要知道它前面的结点，才能修改：

```text
pre->next = p->next;
```

### 11.4 头结点

链表中常使用头结点统一操作。

有头结点的好处：

- 表头插入和普通位置插入操作一致。
- 表头删除和普通位置删除操作一致。
- 空表和非空表处理更统一。

注意：

- 头结点不一定存放有效数据。
- 头指针是指向链表第一个结点的指针。
- 有头结点时，头指针通常指向头结点。

## 12. 易错点

1. 把线性表和顺序表混为一谈。

   线性表是逻辑结构，顺序表是线性表的一种存储实现。

2. 位序和下标混淆。

   第 `i` 个元素对应数组下标 `i - 1`。

3. 认为链表插入删除一定是 `O(1)`。

   如果题目没有直接给出结点指针，通常还需要先查找位置，整体仍可能是 `O(n)`。

4. 忘记链表最后一个结点的 `next` 应为 `NULL`。

5. 删除链表结点后忘记释放内存。

6. 插入链表结点时指针修改顺序错误。

   单链表在结点 `p` 后插入新结点 `s` 时，应先执行：

   ```c
   s->next = p->next;
   p->next = s;
   ```

7. 删除单链表结点时忘记保存后继。

8. 顺序表插入前忘记判断是否已满。

9. 顺序表删除前忘记判断位置是否合法。

## 13. 复习口诀

```text
线性结构一对一，前驱后继要牢记。
顺序存储地址连，随机访问最方便。
链式存储靠指针，插删灵活查找慢。
位序从一数组零，插删边界要看清。
```

- 顺序表的结构、插入、删除、查找。
- 单链表的结构、头插法、尾插法、插入、删除、查找。
- 顺序表和链表的优缺点对比。
- 常见时间复杂度。
- 双指针、前驱指针、头结点等常用思想。
